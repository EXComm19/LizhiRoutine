# Product And Principles

## Product Name

Lizhi Routine

## Product Category

Personal productivity, time blocking, routine planning, deadline tracking, and time-spend analytics.

## Core Idea

Lizhi Routine treats time as an allocated resource. The app combines:

- todos that may have due dates but do not require scheduled times at creation,
- routine templates that can be reused many times,
- fixed calendar events that block unavailable time,
- longer periods such as internships, placements, holidays, or study blocks,
- sleep planning,
- statistics grouped by the original todo or routine source.

The main interaction is placing blocks onto a vertical daily timeline. Scheduling a todo does not consume or delete the todo. A single large assignment can remain one todo while being worked on across many blocks.

## User Mental Model

The user has three kinds of commitments:

- **Intentions:** todos/reminders they want or need to complete.
- **Reusable investments:** routines like gym, commute, shower, deep work, reading, meal prep.
- **Fixed obligations:** calendar imports and period schedules that frame the day.

The timeline is where these become a concrete day plan.

## Tier System

Every todo, routine, and timeline block has a tier:

| Tier | Meaning | Examples |
| --- | --- | --- |
| `T0` | Highest priority / high consequence / deep investment | assignment due soon, exam prep, income-generating work |
| `T1` | Important maintenance or planned growth | lecture, lab, gym, active learning |
| `T2` | Necessary routine/admin/low-return maintenance | commute, chores, errands |

The tier is a label and visual signal. It should not dominate the UI.

## Product Tone

Lizhi Routine should feel:

- calm,
- precise,
- editorial,
- focused,
- warm,
- personal,
- low-noise,
- more like a crafted daily planner than a generic enterprise calendar.

Avoid:

- bright generic productivity blue as the main brand color,
- dense spreadsheet energy,
- large marketing-style heroes,
- cartoonish gamification,
- redundant labels,
- explaining the app inside the app.

## MVP Target User

A student or knowledge worker with:

- deadlines,
- lectures/labs/work periods,
- repeated routines,
- big tasks that require multiple work sessions,
- interest in seeing where time was allocated.

The first iOS MVP should make daily use comfortable on a phone, then sync with the web app through Supabase.
